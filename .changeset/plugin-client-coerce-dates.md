Add a `coerceDates` option that converts date values at runtime so they match the `Date` types generated by `@kubb/plugin-ts` (kubb-labs/kubb#1429). Requires the OpenAPI adapter to run with `dateType: 'date'`.

When enabled, Kubb generates a transformer per schema and applies it inside every operation:

- **Responses** — ISO `string` → `Date` (via `new Date(value)`).
- **Request bodies** — `Date` → ISO `string` (`date` → `YYYY-MM-DD`, `time` → `HH:mm:ss`, `date-time` → full ISO).

Detection is schema-driven: only fields actually typed as a date are converted, following nested objects, arrays, and `$ref`s. Refs delegate to the referenced schema's transformer, so recursive schemas terminate. Applies to `parser: 'client'` with the default `clientType: 'function'`; with `parser: 'zod'`, Zod owns response validation/coercion and `coerceDates` is ignored.
